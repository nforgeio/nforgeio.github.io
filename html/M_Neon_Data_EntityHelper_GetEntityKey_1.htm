Remarks

Stoke follows a common convention where Couchbase entities are persisted using a key formed by appending entity reference to the entity type, separated by a double colon ("::"). This makes entity types available for filtering when managing cross datacenter replication.

This method concatenates the entity type and ID using a ("::") separator.
