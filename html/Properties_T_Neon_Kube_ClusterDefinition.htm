ClusterDefinition Properties

The ClusterDefinition type exposes the following members.

Properties
  NameDescription
AllowUnitTesting
Optionally enable unit testing on this cluster. This is disabled by default for safety.
Ceph
Returns the options to be used for configuring the cluster integrated Ceph file system.
Datacenter
Identifies the datacenter.
Docker
Returns the options to be used when installing Docker on each of the cluster nodes.
DrivePrefix

Returns the prefix for block devices that will be attached to the host machines. For many hosting environments this will be sd, indicating that drives will be attached like: /dev/sda, /dev/sdb, /dev/sdc...

This may be different though for some hosting environments. XenServer for example, uses the xvd prefix and attaches drives as /dev/sda, /dev/sdb, /dev/sdc...

 Note
This property is set automatically during cluster provisioning.
Environment
Indicates how the cluster is being used.
Hosting
Specifies hosting related settings (e.g. the cloud provider). This defaults to null which indicates that the cluster will be hosted on private servers.
Kubernetes
Returns the Kubernetes cluster options.,
Masters
Enumerates the cluster master node definitions.
Mon
Returns the options to be used for configuring the cluster integrated Elasticsearch/Fluentd/Kibana (Mon) logging stack.
Name
The cluster name.
NameRegex
Regex for verifying cluster names for hosts, routes, groups, etc.
Network
Describes the cluster's network configuration.
NodeDefinitions
Describes the host nodes in the cluster.
NodeOptions
Specifies host node options.
Nodes
Enumerates all cluster node definitions.
PackageProxy
Optionally specifies one or more APT proxy/cache servers the cluster will use to install and update Linux packages. These are endpoints like HOSTNAME:PORT or ADDRESS.PORT of a apt-cacher-ng or other package proxy server. The port is generall set to 3142 Multiple proxies may be specified by separating them with spaces. This defaults to referencing the apt-cacher-ng instances running on the master nodes.
Prometheus
Returns the options to be used for configuring the cluster integrated Elasticsearch/Fluentd/Kibana (Mon) logging stack.
Provisioner
Identifies the tool/version used to provision the cluster.
Setup
Optionally specifies setup process related options.
SortedMasters
Enumerates the cluster master node definitions sorted in ascending order by name.
SortedNodes
Enumerates all cluster node definitions sorted in ascending order by name.
SortedWorkers
Enumerates the cluster worker node definitions sorted in ascending order by name.
TimeSources
Specifies the NTP time sources to be configured for the cluster. These are the FQDNs or IP addresses of the sources. This defaults to pool.ntp.org.
Workers
Enumerates the cluster worker node definitions.
