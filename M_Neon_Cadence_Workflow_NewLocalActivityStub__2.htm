Remarks
 Note
Unlike workflow stubs, a single activity stub instance can be used to launch multiple activities.

Activities launched by the returned stub will be executed in the current process. This is intended to easily and efficiently execute activities that will complete very quickly (usually within a few seconds). Local activities are similar to normal activities with these differences:

  • Local activities are always scheduled to executed within the current process.
  • Local activity types do not need to be registered with the worker.
  • Local activities must complete within the DecisionTaskTimeout. This defaults to 10 seconds and can be set to a maximum of 60 seconds.
  • Local activities cannot heartbeat.
