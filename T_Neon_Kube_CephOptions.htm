Enabled
Indicates whether Ceph storage is to be enabled for the cluster. This defaults to true.
MDSCacheSize

Specifies the default amount of RAM to allocate to Ceph MDS processes for caching (ByteUnits). This can be overridden for specific nodes. This defaults to 64Mi.

 Note

The Ceph documentation states that MDS may tend to underestimate the RAM it's using by up to 1.5 times. To avoid potential memory issues, neonKUBE will adjust this value by dividing it by 1.5 to when actually configuring the MDS services.

You should also take care to leave 1-2GB of RAM for the host Linux operating system as well as the OSD non-cache related memory when you're configuring this property.

 Note
The default is probably too small for production environments
OSDCacheSize

Specifies the default amount of RAM to allocate to Ceph OSD processes for caching (ByteUnits). his can be overridden for specific nodes. This defaults to 256Mi.

 Note

The Ceph documentation states that OSD may tend to underestimate the RAM it's using by up to 1.5 times. To avoid potential memory issues, neonKUBE will adjust this value by dividing it by 1.5 to when actually configuring OSD services.

You should also take care to leave 1-2GB of RAM for the host Linux operating system as well as the OSD non-cache related memory when you're configuring this property.

 Note
The default may be too small for production environments
OSDDriveSize

Specifies the default size of the Ceph OSD drives created for cloud and hypervisor based environments (ByteUnits). This can be overridden for specific nodes. This defaults to 16Gi.

 Note
The default may be too small for production environments
OSDJournalSize

Specifies the default size to allocate for the OSD journals (ByteUnits). This can be overridden for specific nodes. This defaults to 1Gi.

 Note
The default may be too small for production environments
OSDObjectSizeMax
Specifies the maximum size of a Ceph RADOS object in bytes (ByteUnits). This can be overridden for specific nodes. This defaults to 5Gi.
OSDPlacementGroups
Specifies the default number of placement groups assigned to each OSD. This defaults to 100.
OSDReplicaCount
Specifies the default number of object replicas to be stored in the cluster. This defaults to the minimum of 3 or the number of OSD nodes provisioned in the cluster.
OSDReplicaCountMin
Specifies the minimum number of objects replicas required when the Ceph storage cluster is operating in a degraded state. This defaults to OSDReplicaCount-1 unless OSDReplicaCount==1 in which case this will also default to 1.
Release
Specifies the Ceph software major release name. This defaults to mimic.
Username
