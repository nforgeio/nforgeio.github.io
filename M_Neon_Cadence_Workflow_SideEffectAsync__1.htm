Remarks

This is similar to what you could do with a local activity but is a bit easier since you don't need to declare the activity and create a stub to call it.

 Note
The function must return within the configured decision task timeout and should avoid throwing exceptions.
 Note
The function passed should avoid throwing exceptions. When an exception is thrown, this method will catch it and simply return the default value for T.
 Note

The .NET version of this method currently works a bit differently than the Java and GOLANG clients which will only call the function once. The .NET implementation calls the function every time SideEffectAsyncT(FuncT) is called but it will ignore the all but the first call's result.

This is an artifact of how the .NET client is currently implemented and may change in the future. You should take care not to code your application to depend on this behavior (one way or the other).
