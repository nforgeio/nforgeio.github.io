Properties
  NameDescription
BackoffCoefficient
Specifies the coefficient used to calculate the next retry backoff interval. The next retry interval is previous interval multiplied by this coefficient. This must be 1 or larger. Default is 2.0.
ExpirationInterval
Maximum time to retry. Either ExpirationInterval or MaximumAttempts is required. Retries will stop when this is exceeded even if maximum retries is not been reached.
InitialInterval
Specifies the backoff interval for the first retry. If coefficient is 1.0 then it is used for all retries. Required, no default value.
MaximumAttempts
Maximum number of attempts. When exceeded the retries stop. If not set or set to 0, it means unlimited, and the policy will rely on ExpirationInterval to decide when to stop retrying. Either MaximumAttempts or MaximumInterval"/> is required.
MaximumInterval
Specifies the maximim retry interval. Retries intervals will start at InitialInterval and then be multiplied by BackoffCoefficient for each retry attempt until the interval reaches or exceeds MaximumInterval, at which point point each retry will use MaximumInterval for all subsequent attempts.
NonRetriableErrors

Specifies Cadence errors that should not trigger a retry. This is optional. Cadence server will stop retrying if error reason matches this list. Use the NonRetriableErrors class methods to initialize this list as required.

 Note
Cancellation is not a failure, so that won't be retried.
