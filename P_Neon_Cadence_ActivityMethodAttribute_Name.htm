ActivityMethodAttributeName Property

Specifies the name to be used to identify a specific activity method. This is optional for activity interfaces that have only one method but is required for interfaces with multiple entry points.

Remarks

When specified, this name will be combined with the activity type name when registering and executing an activity via the method. This will look like:

Copy
ACTIVITY_TYPENNAME::METHODNAME

where ACTIVITY_TYPENNAME is either the activity interface's fully qualified name or the name specified by Name and METHOD_NAME is from Name. This is the same convention implemented by the Java client.

 Note
Some implications of this scheme are that we'll need to register multiple activity types for each activity interface when there are multiple entry points (one per method) and that external activity invocations will need to explicitly specify activity types that include the method name when one is specified to the target method.
