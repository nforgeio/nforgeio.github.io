Parameters

settings (Optional)
Type: Neon.CadenceCadenceSettings
Optional Cadence settings.
image (Optional)
Type: SystemString
Optionally specifies the Cadence container image (defaults to nkubeio/cadence-dev:latest).
name (Optional)
Type: SystemString
Optionally specifies the Cadence container name (defaults to cb-test).
env (Optional)
Type: SystemString
Optional environment variables to be passed to the Cadence container, formatted as NAME=VALUE or just NAME.
defaultDomain (Optional)
Type: SystemString
Optionally specifies the default domain for the fixture's client. This defaults to test-domain.
logLevel (Optional)
Type: Neon.DiagnosticsLogLevel
Specifies the Cadence log level. This defaults to None.
keepConnection (Optional)
Type: SystemBoolean
Optionally specifies that a new Cadence connection should not be established for each unit test case. The same connection will be reused which will save about a second per test.
keepOpen (Optional)
Type: SystemBoolean
Optionally indicates that the container should continue to run after the fixture is disposed.
hostInterface (Optional)
Type: SystemString
Optionally specifies the host interface where the container public ports will be published. This defaults to DefaultHostInterface but may be customized. This needs to be an IPv4 address.
noClient (Optional)
Type: SystemBoolean
Optionally disables establishing a client connection when true is passed. The Client and HttpClient properties will be set to null in this case.
noReset (Optional)
Type: SystemBoolean
Optionally prevents the fixture from calling Reset to put the Cadence client library into its initial state before the fixture starts as well as when the fixture itself is reset.
emulateProxy (Optional)
Type: SystemBoolean
INTERNAL USE ONLY: Optionally starts a partially functional integrated cadence-proxy for low-level testing. Most users should never enable this because it's probably not going to do what you expect.
Remarks
 Note
A fresh Cadence client Client will be established every time this fixture is started, regardless of whether the fixture has already been started. This ensures that each unit test will start with a client in the default state.
