OnePassword Class

Provides access to a developers's 1Password secrets for testing and executing CI/CD purposes.

 Note
This class is not entirely general purpose; it's currently oriented towards supporting CI/CD for neonFORGE maintainers who have configured their workstations by installing both the 1Password application and CLI with the client having been manually signedin for the first time and the NC_USER environment variable set to the 1Password shortname used to sign-in. This also assumes that account's vaults configured a a specific way.

Remarks
 Note
This class requires that the developer has followed the developer onboarding instructions and has already configured the current workstation by installing 1Password etc.

neonFORGE has decided to use the 1Password password manager company wide for password management including devops related activities. All developers will have a neonFORGE 1Password account holding necessary credentials for the individual. All company users will have a 1Password vault named [user-USERNAME] where USERNAME is the person's Microsoft Office user name, such as [sally] from [sally@neonforge.com]. Most, if not all of a person's credentials will be stored in this vault.

Each developer will have a standard set of passwords with well-defined names such that they can be accessed by common CI/CD scripts, etc.

We're currently persisting credentials available for automation as 1Password passwords. For real secrets, we'll persist the value using the [password] field within the 1Password password. For information that is not really a secret (and may trigger 1Password complexity warnings), we use the [value] field.

Use GetPassword(String, String) to retrieve a password and GetValue(String, String) to retrieve a value for the current user.

 Note
The current user is determined by the NC_USER environment variable. This is set to the user's neonFORGE Microsoft Office user name when the workstation is configured by the developer.
 Note
This class mimics the PowerShell 1Password script functions found in $/Powershell/includes.ps1 so C# code can also access secrets.
 Note
This class can interoperate with the 1Password PowerShell script functions.
