ExceptionResult Class

Remarks
 Note
This is typically be used internally by frameworks like Neon.Cadence to bring exception handling functionality when calling remote processes (like workflows and activities in that case).

This works by setting the ExceptionType property to the fully qualified name of the exception type and ExceptionMessage to the exception message (which may be null) in the remote process and then serializing the instance to JSON or some other format to be transmitted back to the caller.

The local caller will then call ThrowOnError which will attempt to rethrow the exception with the same ExceptionType. This is accomplished by reflecting all assemblies currently loaded in the AppDomain looking for exception types. Exception types that have a default constructor or constructors with a string message parameter or a message and inner exception message parameters will be cached internally in a dictionary that maps the type name to the exception type.

If ThrowOnError can locate a local exception type that has a conforming constructor, then that exception type will be thrown with the ExceptionMessage. A CatchAllException will be thrown instead, if a conforming local exception doesn't exist.
