NodeDefinition Properties

The NodeDefinition type exposes the following members.

Properties
Â  NameDescription
Azure
Azure provisioning options for this node, or null to use reasonable defaults.
IsMaster
Indicates that the node will act as a master node (defaults to false).
IsWorker
Returns true for worker nodes.
Labels
Specifies the labels to be assigned to the host node. These can provide detailed information such as the host CPU, RAM, storage, etc. NodeLabels for more information.
Name
Uniquely identifies the node within the cluster.
PrivateAddress
The node's IP address or null if one has not been assigned yet. Note that an node's IP address cannot be changed once the node has been added to the cluster.
PublicAddress
The node's public IP address or DNS name. This will be generally initialized to null before provisioning a cluster. This will be initialized while by the neon-cli tool for master nodes when provisioning in a cloud provider.
Role
Returns the node's NodeRole. This defaults to Worker.
Taints
Specifies the taints to be assigned to the host node.
VmDisk
The amount of disk space to allocate to this node when when provisioned on a hypervisor. This is specified as a string that can be a byte count or a number with units like 512MB, 0.5GB, 2GB, or 1TB. This defaults to the value specified by VmDisk.
VmHost
Identifies the hypervisor instance where this node is to be provisioned for Hyper-V or XenServer based clusters. This name must map to the name of one of the VmHosts when set.
VmMemory
Specifies the maximum amount of memory to allocate to this node when provisioned on a hypervisor. This is specified as a string that can be a byte count or a number with units like 512MB, 0.5GB, 2GB, or 1TB. This defaults to the value specified by VmMemory.
VmProcessors
Specifies the number of processors to assigned to this node when provisioned on a hypervisor. This defaults to the value specified by VmProcessors.
