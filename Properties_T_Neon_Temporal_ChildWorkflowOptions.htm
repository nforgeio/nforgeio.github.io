ChildPolicy
Optionally specifies what happens to the child workflow when the parent is terminated. This defaults to RequestCancel.
CronSchedule
Optionally specifies a recurring schedule for the workflow. This can be set to a string specifying the minute, hour, day of month, month, and day of week scheduling parameters using the standard Linux CRON format described here: https://en.wikipedia.org/wiki/Cron
DecisionTaskTimeout
Optionally specifies the decision task timeout for the child workflow. This defaults to WorkflowDecisionTaskTimeout.
Namespace
Optionally specifies the target namespace. This defaults to the namespace specified by Namespace, Namespace, or to the client's default namespace as specified by Namespace (in that order of precedence).
RetryOptions
Optionally specifies retry options.
ScheduleToStartTimeout
Optionally specifies the default maximum time a workflow can wait between being scheduled and actually begin executing. This defaults to 24 hours.
StartToCloseTimeout
Specifies the maximum time the child workflow may execute from start to finish. This defaults to WorkflowStartToCloseTimeoutSeconds.
TaskQueue
Optionally specifies the Temporal task queue where the child workflow will be scheduled. This defaults to the namespace specified by TaskQueue or TaskQueue, or the parent workflow's domain, in that order of precedence.
WaitUntilFinished
Optionally specifies whether to wait for the child workflow to finish for any reason including being: completed, failed, timedout, terminated, or canceled.
WorkflowId
Optionally specifies the workflow ID to assign to the child workflow. A UUID will be generated by default.
WorkflowIdReusePolicy
Optionally determines how Temporal handles workflows that attempt to reuse workflow IDs. This generally defaults to AllowDuplicate but the default can be customized via the WorkflowMethodAttribute tagging the workflow entry point method or WorkflowIdReusePolicy (which also defaults to AllowDuplicate.
