CadenceFixtureStart Method (CadenceSettings, String, String, String, String, LogLevel, Boolean, Boolean, String, Boolean, Boolean, ContainerLimits)

Starts a Cadence container if it's not already running. You'll generally want to call this in your test class constructor instead of Start(Action).

 Note
You'll need to call StartAsComposed(CadenceSettings, String, String, String, String, LogLevel, Boolean, Boolean, String, Boolean, Boolean, ContainerLimits) instead when this fixture is being added to a ComposedFixture.

Optional environment variables to be passed to the Cadence container, formatted as NAME=VALUE or just NAME.

Optionally specifies the default domain for the fixture's client. This defaults to test-domain.
logLevel (Optional)
Type: Neon.DiagnosticsLogLevel
Specifies the Cadence log level. This defaults to None.
reconnect (Optional)
Type: SystemBoolean
Optionally specifies that a new Cadence connection should be established for each unit test case. By default, the same connection will be reused which will save about a second per test.
keepRunning (Optional)
Type: SystemBoolean
Optionally indicates that the container should remain running after the fixture is disposed. This is handy for using the Temporal web UI for port mortems after tests have completed.
hostInterface (Optional)
Type: SystemString
Optionally specifies the host interface where the container public ports will be published. This defaults to DefaultHostInterface but may be customized. This needs to be an IPv4 address.
noClient (Optional)
Type: SystemBoolean
Optionally disables establishing a client connection when true is passed. The Client and HttpClient properties will be set to null in this case.
noReset (Optional)
Type: SystemBoolean
Optionally prevents the fixture from calling Reset to put the Cadence client library into its initial state before the fixture starts as well as when the fixture itself is reset.
limits (Optional)
Type: Neon.XunitContainerLimits
Specifies the Docker container limits to use for hosting Couchbase. Note that this method will use reasonably small default limits when this is null.

Return Value

Type: TestFixtureStatus
Started if the fixture wasn't previously started and this method call started it or AlreadyRunning if the fixture was already running.
Remarks
 Note
Some of the settings properties will be ignored including Servers. This will be replaced by the local endpoint for the Cadence container. Also, the fixture will connect to the default Cadence domain by default (unless another is specified).
 Note
A fresh Cadence client Client will be established every time this fixture is started, regardless of whether the fixture has already been started. This ensures that each unit test will start with a client in the default state.
