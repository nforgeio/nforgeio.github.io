VmDisk
Specifies the maximum amount of memory to allocate to each cluster virtual machine. This is specified as a string that can be a long byte count or a byte count or a number with units like 512MiB, 0.5GiB, 2GiB, or 1TiB. This defaults to 64GiB.
VmDriveFolder

Path to the folder where virtual machine hard drive folders are to be persisted. This defaults to the local Hyper-V folder for Windows.

 Note
This is recognized only when deploying on a local Hyper-V hypervisor, typically for development and test purposes. This is ignored when provisioning on remote Hyper-V instances or for cloud or bare machine environments.
VmHostPassword
The default password to use for connecting the hypervisor host machines specified by VmHosts. This may be overridden for specific hypervisor machines within VmHosts items. This defaults to null.
VmHosts
Optionally identifies the target Hyper-V or XenServer hypervisor machines.
VmHostUsername

The default username to use for connecting the hypervisor host machines specified by VmHosts. This may be overridden for specific hypervisor machines. This defaults to null.

 Note
This defaults to root for XenServer based environments.
VmMemory
Specifies the default maximum amount of memory to allocate to each cluster virtual machine. This is specified as a string that can be a byte count or a number with units like 512MiB, 0.5GiB, 2iGB, or 1TiB. This defaults to 4GiB.
VmNamePrefix

The prefix to be prepended to virtual machine provisioned to hypervisors for the HyperV, HyperVLocal, and XenServer environments.

When this is null (the default), the cluster name followed by a dash will prefix the provisioned virtual machine names. When this is a non-empty string, the value followed by a dash will be used. If this is empty or whitespace, machine names will not be prefixed.

 Note
Virtual machine name prefixes will always be converted to lowercase.
VmProcessors
The default number of virtual processors to assign to each cluster virtual machine.
XenServer
Specifies the hosting settings when hosting on Citrix XenServer or the XCP-ng hypervisors.
