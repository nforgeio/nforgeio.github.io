RunOptions Enumeration

Enumerates the possible options to use when executing a remote command on a LinuxSshProxyTMetadata. These options may be combined using the bitwise OR operator.

Members
  Member nameValueDescription
None0 No options are set.
Defaults1 Bitwise ORs any specific option flags with those specified by DefaultRunOptions. This is handy for setting or resetting flags like FaultOnError on a global basis for a node proxy instance.
FaultOnError2 Puts the LinuxSshProxyTMetadata into the faulted state when the command returns a non-zero exit code.
RunWhenFaulted4 Runs the command even if the LinuxSshProxyTMetadata is in the faulted state.
IgnoreRemotePath8 Ignore the RemotePath property.
BinaryOutput16 Return the standard output from remote command as binary data rather than intrepreting it as text.
Redact32 Use for commands that may include sensitive secrets as command arguments and/or results. Only limited information about commands run with this flag will be logged.
LogOnErrorOnly64 Logs command output only if the command returns a non-zero exit code.
LogOutput128 Logs the command standard output (standard error output is logged by default).
LogBundle256 Used internally to prevent logging of the command "START: *" line at lower levels because this has already been logged.
Audit512 Used to mark commands whose execution should be logged for auditing.
Shutdown1024

Indicates that the command will shutdown or restart or reboot the target server. Commands with this flag will not be retried on the server.

 Note
IMPORTANT: You must specify this flag if your command shutdown down the machine to prevent causing an infinite reboot loop.
