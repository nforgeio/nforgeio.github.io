NetworkOptions Properties

The NetworkOptions type exposes the following members.

Properties
  NameDescription
Cni
Specifies the cluster network provider. This currently defaults to Calico but this will change to the Istio integrated provider once that stablizes.
CniVersion

Specifies the cluster network provider version. This defaults to default which will install a reasonable supported version.

 Note
This is ignored for the Istio integrated provider.
Gateway
Specifies the default network gateway address to be configured for hosts. This defaults to the first usable address in the PremiseSubnet. For example, for the 10.0.0.0/24 subnet, this will be set to 10.0.0.1. This is ignored for cloud hosting environments.
Ingress
Optionally sets the ingress ports.
IstioVersion
Specifies the version of Istio to be installed. This defaults to default which will install a reasonable supported version.
MutualPodTLS
Optionally enable Istio mutual TLS support for cross pod communication. This defaults to false.
Nameservers
The IP addresses of the upstream DNS nameservers to be used by the cluster. This defaults to the Google Public DNS servers: [ "8.8.8.8", "8.8.4.4" ] when the property is null or empty.
NodeSubnet

The subnet where the cluster nodes reside.

 Note
This property must be configured for the on-premise providers (Machine, HyperV, and XenServer". This is computed automatically by the neon tool when provisioning in a cloud environment.
 Note
For on-premise clusters, the statically assigned IP addresses assigned to the nodes must reside within the this subnet. The network gateway will be assumed to be the second address in this subnet and the broadcast address will assumed to be the last address.
PodSubnet

Specifies the pod subnet to be used for the cluster. This subnet will be split so that each node will be allocated its own subnet. This defaults to 10.254.0.0/16.

 Note
WARNING: This subnet must not conflict with any other subnets provisioned within the premise network.
PremiseSubnet
Specifies the subnet for entire host network for on-premise environments like Machine, HyperVLocal and XenServer. This is required for those environments.
ServiceSubnet
Specifies the subnet subnet to be used for the allocating service addresses within the cluster. This defaults to 10.253.0.0/16.
