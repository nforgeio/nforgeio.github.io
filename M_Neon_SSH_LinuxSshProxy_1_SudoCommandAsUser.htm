Remarks
 Note
command may not include single quotes or redirect angle brackets such as < or >>. For more complex command, try uploading and executing a CommandBundle instead.

The runOptions flags control how this command functions. If FaultOnError is set, then commands that return a non-zero exit code will put the server into the faulted state by setting IsFaulted=true. This means that IsReady will always return false afterwards and subsequent command executions will be ignored unless RunWhenFaulted is specified for the future command.

LogOnErrorOnly indicates that command output should be logged only for non-zero exit codes.

 Note
Any null arguments will be ignored.
