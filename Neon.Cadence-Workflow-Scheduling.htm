Timeouts

Cadence enforces three timouts for scheduled workflows. These can be customized when scheduling a workflow via WorkflowOptions and ChildWorkflowOptions.

ScheduleToStartTimeout

This limits the time a workflow can wait to be assigned for execution on a worker instance. This can be used as a way to abort workflows that must execute within a period of time. This defaults to 24 hours for the .NET Cadence client.

StartToCloseTimeout

This limits the time a workflow can execute on a worker instance. This can be used as a way to abort workflows that must execute within a period of time. This defaults to 24 hours for the .NET Cadence client.

DecisionTaskTimeout

This is the decision task timeout. This limits the time a workflow may take to execute any decision logic. This defaults to 10 seconds and may be increased to a maximum of 60 seconds.
