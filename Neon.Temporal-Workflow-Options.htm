Options

You can customize how workflows are started by passing WorkflowOptions when creating an external workflow stub or a ChildWorkflowOptions when creating a child workflow stub. These types are similar and you can use them to customize the workflow ID, timeouts, retry options, etc.

The code sample below starts a workflow with a specific ID and also specifies how long the workflow will be allowed to run:

C#
Copy
// Invoke a workflow with options:

var stub = client.NewWorkflowStub<IHelloWorkflow>(
    new WorkflowOptions()
    {
         WorkflowId             = "my-ultimate-workflow",
         ScheduleToStartTimeout = TimeSpan.FromMinutes(5) 
    });

Console.WriteLine(await stub.HelloAsync("Jeff"));

You can also specify default workflow options for a TemporalClient via TemporalSettings or via the WorkflowInterfaceAttribute tagging your workflow interface. Here's how the .NET client processes these:

  1. WorkflowOptions or ChildWorkflowOptions passed when starting a workflow take precedence over all other options.

  2. Options specified via the WorkflowInterface attribute are recognized next.

  3. Client options specified by TemporalSettings have the lowest precedence.
