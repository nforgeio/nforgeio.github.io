Remarks

Unlike activity stubs, a workflow stub may only be used to launch a single workflow. You'll need to create a new stub for each workflow you wish to invoke and then the first method called on a workflow stub must be the one of the methods tagged by WorkflowMethodAttribute.

workflowTypeName specifies the target workflow implementation type name and optionally, the specific workflow method to be called for workflow interfaces that have multiple methods. For workflow methods tagged by [WorkflowMethod] with specifying a name, the workflow type name will default to the fully qualified interface type name or the custom type name specified by Name.

For workflow methods with Name specified, the workflow type will look like:

Copy
WORKFLOW-TYPE-NAME::METHOD-NAME

You'll need to use this format when calling workflows using external untyped stubs or from other languages. The Java Cadence client works the same way.
