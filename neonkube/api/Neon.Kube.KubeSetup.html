    ConnectCluster(ISetupController)

    Connects to a Kubernetes cluster if it already exists. This sets the K8sClient property in the setup controller state when Kubernetes is running and a connection has not already been established.

    note

    The K8sClient will not be set when Kubernetes has not been started, so Get<TValue>(String) calls for this property will fail when the cluster has not been connected yet, which will be useful for debugging setup steps that require a connection but this hasn't happened yet.

    CreateClusterPrepareController(ClusterDefinition, Boolean, String, String, Int32, IEnumerable<IPEndPoint>, Boolean, Boolean, String, Boolean, Boolean, Boolean, Boolean)

    Constructs the ISetupController to be used for preparing a cluster.

    Declaration
    public static ISetupController CreateClusterPrepareController(ClusterDefinition clusterDefinition, bool cloudMarketplace, string nodeImageUri = null, string nodeImagePath = null, int maxParallel = 0, IEnumerable<IPEndPoint> packageCacheEndpoints = null, bool unredacted = false, bool debugMode = false, string baseImageName = null, bool removeExisting = false, bool disableConsoleOutput = false, bool desktopImage = false, bool desktopReadyToGo = false)
    Parameters
    Type Name Description
    ClusterDefinition clusterDefinition

    The cluster definition.
    Boolean cloudMarketplace

    For cloud environments, this specifies whether the cluster should be provisioned using a VM image from the public cloud marketplace when true or from the private NEONFORGE image gallery for testing when false. This is ignored for on-premise environments.

    note

    Only NEONFORGE maintainers will have permission to use the private image.

    String nodeImageUri

    Optionally specifies the node image URI.

    String nodeImagePath

    Optionally specifies the node image path.

    note

    One of nodeImageUri or nodeImagePath must be specified for on-premise hypervisor based environments. These are ignored for cloud hosting.

    Int32 maxParallel

    Optionally specifies the maximum number of node operations to be performed in parallel. This defaults to 0 which means that we'll use MaxParallel.
    IEnumerable<IPEndPoint> packageCacheEndpoints

    Optionally specifies the IP endpoints for the APT package caches to be used by the cluster, overriding the cluster definition settings. This is useful when package caches are already deployed in an environment.

    note

    Package cache servers are deployed to the control-plane nodes by default.

    Boolean unredacted

    Optionally indicates that sensitive information won't be redacted from the setup logs (typically used when debugging).
    Boolean debugMode

    Optionally indicates that the cluster will be prepared in debug mode.
    String baseImageName

    Optionally specifies the base image name to use for debug mode.
    Boolean removeExisting

    Optionally remove any existing cluster with the same name in the target environment.
    Boolean disableConsoleOutput

    Optionally disables status output to the console. This is typically enabled for non-console applications.
    Boolean desktopImage

    Optionally indicates that we're building a ready-to-go neon desktop image.
    Boolean desktopReadyToGo

    Optionally indicates that we're setting up a neon-desktop built-in cluster from a completely prebuilt desktop image. In this case, the controller returned will fully deploy the cluster (so no setup controller needs to be created and run afterwards).
    Returns
    Type Description
    ISetupController

    The ISetupController.
