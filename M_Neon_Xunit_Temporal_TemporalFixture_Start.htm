TemporalFixtureStart Method (TemporalSettings, String, String, String, LogLevel, Boolean, Boolean, Boolean, Boolean)

Starts a Temporal compose application if it's not already running. You'll generally want to call this in your test class constructor instead of Start(Action).

 Note
You'll need to call StartAsComposed(TemporalSettings, String, String, String, LogLevel, Boolean, Boolean, Boolean, Boolean) instead when this fixture is being added to a ComposedFixture.

Parameters

settings (Optional)
Type: Neon.TemporalTemporalSettings
Optional Temporal settings.
composeFile (Optional)
Type: SystemString

Optionally specifies the Temporal Docker compose file text. This defaults to DefaultComposeFile which configures Temporal server to start with a new Cassandra database instance listening on port 9042 as well as the Temporal web UI running on port 8088. Temporal server is listening on its standard gRPC port 7233.

You may specify your own Docker compose text file to customize this by configuring a different backend database, etc.

name (Optional)
Type: SystemString
Optionally specifies the Docker compose application name (defaults to temporal-dev).
defaultNamespace (Optional)
Type: SystemString
Optionally specifies the default namespace for the fixture's client. This defaults to test-namespace.
logLevel (Optional)
Type: Neon.DiagnosticsLogLevel
Specifies the Temporal log level. This defaults to None.
reconnect (Optional)
Type: SystemBoolean
Optionally specifies that a new Temporal connection should be established for each unit test case. By default, the same connection will be reused which will save about a second per test.
keepRunning (Optional)
Type: SystemBoolean
Optionally indicates that the application should remain running after the fixture is disposed. This is handy for using the Temporal web UI for port mortems after tests have completed.
noClient (Optional)
Type: SystemBoolean
Optionally disables establishing a client connection when true is passed. The Client and HttpClient properties will be set to null in this case.
noReset (Optional)
Type: SystemBoolean
Optionally prevents the fixture from calling Reset to put the Temporal client library into its initial state before the fixture starts as well as when the fixture itself is reset.

Return Value

Type: TestFixtureStatus
Started if the fixture wasn't previously started and this method call started it or AlreadyRunning if the fixture was already running.
Remarks
 Note
Some of the settings properties will be ignored including HostPort. This will be replaced by the local endpoint for the Temporal application. Also, the fixture will connect to the default Temporal namespace by default (unless another is specified).
 Note
A fresh Temporal client Client will be established every time this fixture is started, regardless of whether the fixture has already been started. This ensures that each unit test will start with a client in the default state.
