ServiceDescription Properties

The ServiceDescription type exposes the following members.

Properties
  NameDescription
Address
When set, this overrides Name, Namespace, and Domain when generating the Hostname result. This is typically set for public service endpoints for public deployments or when emulating a cluster on a local machine for testing purposes. This defaults to null.
Domain
The cluster's configured domain (aka zone). This defaults to "cluster.local".
Endpoints
The service's network endpoints.
FullHostname

Returns the hostname to be used to communcate with this service. When deployed to a Kubernetes cluster, this will be formed from Name and Namespace, including the Domain. When testing and Address is not null, then this will simply be the address converted to a string.

 Note
Use Hostname if you need the fully qualified hostname that includes the cluster domain.
Hostname

Returns the hostname to be used to communcate with this service. When deployed to a Kubernetes cluster, this will be formed from Name and Namespace, omitting the Domain. When testing and Address is not null, then this will simply be the address converted to a string.

 Note
Use FullHostname if you need the fully qualified hostname that includes the cluster domain.
Name
The service name as deployed to Kubernetes.
Namespace
The Kubernetes namespace where the service is deployed. This defaults to "default".
