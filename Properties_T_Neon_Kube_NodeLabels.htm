NodeLabels Properties

The NodeLabels type exposes the following members.

Properties
  NameDescription
All
Enumerates all node labels.
CephMDS
io.neonkube.ceph.mds [bool]: Indicates that a Ceph MDS (metadata server) will be deployed to this node if Enabled is true. This defaults to false.
CephMDSCacheSize

io.neonkube.ceph.mds_cachesize_mb [int]: Specifies the RAM in bytes ByteUnits) to assign to the Ceph MDS services for caching if the integrated Ceph storage cluster is enabled and CephMDS is true for this node. This defaults to MDSCacheSize (1GB) (which is probably too small for production clusters).

 Note

The Ceph documentation states that OSDs may tend to underestimate the RAM it's using by up to 1.5 times. To avoid potential memory issues, neonKUBE will adjust this value by dividing it by 1.5 to when actually configuring the MDS services.

You should also take care to leave 1-2GB of RAM for the host Linux operating system as well as the OSD non-cache related memory when you're configuring this property.

CephMGR
io.neonkube.ceph.mgr [bool]: Indicates that a Ceph MGR (manager server) will be deployed to this node if Enabled is true. This defaults to false.
CephMON
io.neonkube.ceph.monitor [bool]: Indicates that the Ceph monitor and manager services will be deployed to this node if Enabled is true. This defaults to false.
CephOSD
io.neonkube.ceph.osd [bool]: Indicates that a Ceph OSD (object storage daemon) will be deployed to this node if Enabled is true. This defaults to false.
CephOSDCacheSize

io.neonkube.ceph.cachesize [int]: Specifies the RAM in bytes ByteUnits) to assign to the Ceph OSDs for caching if the integrated Ceph storage cluster is enabled and CephOSD is true for this node. This defaults to OSDCacheSize (1GB) (which is probably too small for production clusters).

 Note

The Ceph documentation states that OSDs may tend to underestimate the RAM it's using by up to 1.5 times. To avoid potential memory issues, neonKUBE will adjust this value by dividing it by 1.5 to when actually configuring the OSD services.

You should also take care to leave 1-2GB of RAM for the host Linux operating system as well as the OSD non-cache related memory when you're configuring this property.

CephOSDDevice
io.neonkube.ceph.osd_device (string]: The path to the block device where the OSD data will be persisted (like: /dev/sdb) when CephOSD is true. This will be initialized automatically for most hosting environments but will need to be specified explicitly for Machine.
CephOSDDriveSize
io.neonkube.ceph.drivesize [int]: Specifies the size in bytes (ByteUnits) of the Ceph OSD drive created for cloud and hypervisor based environments if the integrated Ceph storage cluster is enabled and CephOSD is true for this node. This defaults to OSDDriveSize (128GiB).
CephOSDJournalSize
io.neonkube.ceph.journalsize [int]: Specifies the disk capacity in bytes ByteUnits) to assign to the Ceph OSD journal if the integrated Ceph storage cluster is enabled and CephOSD is true for this node. This defaults to OSDJournalSize.
ComputeCores
io.neonkube/compute.cores [int]: Specifies the number of CPU cores. This defaults to 0 for Machine and is initialized for cloud and Hypervisor based hosting environments.
ComputeRam
io.neonkube/compute.ram_mb [int]: Specifies the available RAM in megabytes. This defaults to 0 for Machine and is initialized for cloud and Hypervisor based hosting environments.
Custom
Custom node labels.
Elasticsearch
io.neonkube.mon.elasticsearch.enabled [bool]: Indicates that Elasticsearch will be deployed to this node if Enabled is true. This defaults to false.
Istio
io.neonkube.mon.istio.enabled [bool]: Indicates that Istio will be deployed to this node. This defaults to false.
M3DB
io.neonkube.mon.m3db.enabled [bool]: Indicates that M3DB will be deployed to this node if Enabled is true. This defaults to false.
PhysicalFaultDomain
io.neonkube/physical.faultdomain [string]: A free format string grouping the host by the possibility of underlying hardware or software failures. This defaults to the empty string.
PhysicalLocation
io.neonkube/physical.location [string]: A free format string describing the physical location of the server. This defaults to the empty string.
PhysicalMachine
io.neonkube/physical.model [string]: A free format string describing the physical server computer model (e.g. Dell-PowerEdge-R220). This defaults to the empty string.
PhysicalPower
io.neonkube/physical.power [string]: Describes host the physical power to the server may be controlled. This defaults to the empty string.
Standard
Enumerates the node labels.
StorageEphemeral
io.neonkube/storage.redundant [bool]: Indicates that the storage is ephemeral. All data will be lost when the host is restarted. This defaults to false for on-premise clusters and is computed for cloud deployments.
StorageHDD
io.neonkube/storage.hdd [bool]: Indicates that the storage is backed by a spinning drive as opposed to a SSD. This defaults to false for on-premise clusters and is computed for cloud deployments.
StorageLocal
io.neonkube/storage.local [bool]: Specifies whether the node storage is hosted on the node itself or is mounted as a remote file system or block device. This defaults to true for on-premise clusters and is computed for cloud deployments.
StorageRedundant
io.neonkube/storage.redundant [bool]: Indicates that the storage is redundant. This may be implemented locally using RAID1+ or remotely using network or cloud-based file systems. This defaults to false for on-premise clusters and is computed for cloud deployments.
StorageSize
io.neonkube/storage.size [string]: Specifies the node primary drive storage capacity in bytes (ByteUnits).
