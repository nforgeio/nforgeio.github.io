AzureOptionsPublicNodeAddresses Property

 Note
IMPORTANT: assigning public IP addresses to cluster nodes is not currently implemented.

Specifies whether the cluster nodes should be provisioned with public IP addresses in addition to the cluster wide public IP addresses assigned to the traffic manager. This defaults to false.

 Note
You will incur additional recuring costs for each public IP address.

Remarks

There are two main reasons for enabling this.

  1. Outbound SNAT port exhaustion: This can occur when cluster nodes behind a load balancer have a high rate of outbound requests to the Internet. The essential issue is that the traffic manager can NAT a maximum of 64K outbound connections for the entire cluster. This is described in detail here. Assigning a public IP address to each node removes this cluster level restriction such that each node can have up to 64K outbound connections.
  2. Occasionally, it's important to be able to reach specific cluster nodes directly from the Internet.

Enabling this directs the neon-cli to create a dynamic instance level IP address for each cluster node and add a public network interface to each cluster virtual machine.

 Note
The public network interface will be protected by a public security group that denies all inbound traffic and allows all outbound traffic by default.
