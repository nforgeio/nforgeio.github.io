Properties
Â  NameDescription
ChildPolicy
Optionally specifies what happens to the child workflow when the parent is terminated. This defaults to Abandon.
CronSchedule
Optionally specifies a recurring schedule for the workflow. This can be set to a string specifying the minute, hour, day of month, month, and day of week scheduling parameters using the standard Linux CRON format described here: https://en.wikipedia.org/wiki/Cron
Domain
Specifies the task list where the child workflow will be scheduled.
RetryOptions
Optionally specifies retry options.
ScheduleToCloseTimeout
Specifies the maximum time the child workflow may execute from start to finish. This defaults to WorkflowScheduleToCloseTimeoutSeconds.
ScheduleToStartTimeout
Optionally specifies the default maximum time a workflow can wait between being scheduled and actually begin executing. This defaults to 24 hours.
TaskList
Optionally specifies the task list where the child workflow will be scheduled. This defaults to the parent's task list.
TaskStartToCloseTimeout
Optionally specifies the decision task timeout for the child workflow. This defaults to WorkflowTaskStartToCloseTimeout.
WaitUntilFinished
Optionally specifies whether to wait for the child workflow to finish for any reason including being: completed, failed, timedout, terminated, or canceled.
WorkflowId
Optionally specifies the workflow ID to assign to the child workflow. A UUID will be generated by default.
WorkflowIdReusePolicy
Optionally determines how Cadence handles workflows that attempt to reuse workflow IDs. This generally defaults to AllowDuplicateFailedOnly but the default can be customized via the WorkflowMethodAttribute tagging the workflow entry point method or WorkflowIdReusePolicy (which defaults to AllowDuplicateFailedOnly.
