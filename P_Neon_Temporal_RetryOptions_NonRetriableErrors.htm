RetryOptionsNonRetriableErrors Property

Specifies Temporal errors that should not trigger a retry. This is optional. Temporal server will stop retrying if error reason matches this list. Use the NonRetriableErrors class methods to initialize this list as required.

 Note
Cancellation is not a failure, so that won't be retried.

Remarks

You can specify non-retryable error reasons directly here or use the DoNotRetry(String) method to append a specific reason string or DoNotRetryExceptionType to specify the error reason for an exception type.

We recommend that you use DoNotRetryExceptionType most of the time and reserve DoNotRetry(String) for interop situations where you need to integrate with something written in another language.

For native Temporal exceptions like TemporalTimeoutException, DoNotRetryExceptionType is smart enough to append the proper error reason. For other exception types, this method will use the fully qualified exception type name as the reason.
